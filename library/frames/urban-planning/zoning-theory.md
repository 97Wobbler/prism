# Zoning Theory

## Overview

Zoning is the primary legal mechanism by which municipalities regulate land use and built form. This frame sorts zoning ordinances into four categories based on their **regulatory logic and what they control** — not their surface appearance, but their underlying **theory of what drives development outcomes**. Different zoning types achieve different objectives and require different kinds of enforcement, design review, and incentive structures. Applying the wrong zoning type to a context (e.g., imposing strict Euclidean separation in a dense urban neighborhood, or expecting Performance zoning without adequate design capacity) is a systematic source of development failure and unintended equity consequences.

## Categories

1. **Euclidean Zoning** (conventional use-separation zoning)
   - Controls land use through **enumerated, segregated use categories** (residential, commercial, industrial) with explicit prohibition of mixing.
   - Assumes that separating incompatible uses prevents nuisance and protects property values; relies on district-level density and setback rules.
   - Discriminating criterion: the ordinance defines what uses are *permitted* (as a list) in each zone, not what form or outcomes are acceptable. Non-conforming uses are grandfathered but not allowed new development.
   - Example: single-family zoning that prohibits multifamily housing, neighborhood commercial, or offices in residential districts; industrial zones separated from residential by buffer zones.

2. **Form-based Code** (context-sensitive form regulation)
   - Controls **physical form and urban design** (street frontage, building mass, setbacks, fenestration) more strictly than use.
   - Assumes that the **relationship between building and public realm** is the primary driver of livability and walkability; allows mixed use within a form envelope.
   - Discriminating criterion: the code regulates building facade composition, street-facing requirement, and parking location; use restrictions are minimal or secondary to form compliance.
   - Example: a downtown district code that requires street-level retail frontage, active ground floor, upper-story mixed use, and rear parking — independent of specific tenant type.

3. **Inclusionary Zoning** (use-based affordability coupling)
   - Mandates or incentivizes inclusion of **affordable housing units within market-rate developments**.
   - Assumes that affordability does not happen without explicit requirement or incentive; controls tenure and price through percentage thresholds or density bonuses.
   - Discriminating criterion: the ordinance ties the right to build at certain density or with certain uses to the provision of below-market-rate units. Non-compliance is enforced via deed restriction or community land trust.
   - Example: requiring 15% of units in a residential project to be affordable at 80% AMI, with enforcement through 30-year deed restriction; or offering a density bonus in exchange for on-site affordable units.

4. **Performance Zoning** (outcome-based regulation)
   - Regulates development through **measurable performance standards** (traffic impact, stormwater runoff, noise, lot coverage) rather than explicit use or form permission.
   - Assumes that **outcomes matter more than inputs**; allows mixed, dense, or unconventional development if performance thresholds are met.
   - Discriminating criterion: the ordinance does not enumerate permitted uses; instead, a use is allowed if it meets numerical or modeled performance tests (e.g., "traffic impact < 10% peak-hour increase," "impervious surface ≤ 50%").
   - Example: a mixed-use district that allows any use (retail, office, light industrial, residential) on the same lot if traffic generation, noise, and emissions stay within district caps, verified by traffic study or emissions modeling.

## Classification Procedure

1. Obtain a copy of the zoning ordinance or development code for the area in question.
2. Ask: **"Does the code list specific uses that are permitted vs. prohibited in each zone?"**
   - If **yes, with clear use segregation** (residential zones ban commercial, commercial zones limit residential), go to step 3.
   - If **no, or uses are allowed flexibly**, go to step 4.
3. Ask: **"Are there explicit affordability mandates or density bonuses tied to inclusion of below-market units?"**
   - If **yes** → **Inclusionary Zoning** (possibly overlaid on Euclidean base).
   - If **no** → **Euclidean Zoning**.
4. Ask: **"Does the code emphasize physical form (setbacks, frontage, massing) and allow mixed use within a form envelope?"**
   - If **yes** → **Form-based Code**.
   - If **no**, go to step 5.
5. Ask: **"Does the code regulate through measurable performance standards (traffic, environmental thresholds) rather than use categories?"**
   - If **yes** → **Performance Zoning**.
   - If **no**, return to step 2 — the ordinance may be hybrid or unclassifiable without expert review.
6. State the classification and note any **overlay districts** or **hybrid structures** (e.g., "Euclidean base with inclusionary overlay in transit corridors").

## Implications per Category

| Category | Control mechanism | Development outcome | Enforcement focus |
|---|---|---|---|
| **Euclidean** | Use list; density/setback rules | Separated uses, lower-density, car-dependent | Complaint-driven variance and code enforcement. |
| **Form-based** | Building form; street relationship | Mixed use, walkable, context-aligned form | Design review; subjective compliance with form standards. |
| **Inclusionary** | Affordability mandate or incentive | Market-rate + below-market units colocated | Deed restriction, community land trust, long-term monitoring. |
| **Performance** | Quantified thresholds (traffic, environment) | Mixed, dense, flexible use if thresholds met | Modeling review, compliance verification (traffic study, stormwater calc), post-development auditing. |

Implicit implementation rule: **Euclidean zones require variance processes; form-based codes require skilled design reviewers; inclusionary zoning requires long-term affordability governance; performance zoning requires technical modeling capacity and enforcement rigor.**

## Common Misclassifications

- **Euclidean mistaken for Form-based.** A zoning code adds height limits and facade guidelines to an otherwise use-segregated ordinance, but does not relax use restrictions or allow mixed use. The tell is that the code still separates residential from commercial by district, even if it specifies form detail. Consequence: design review becomes performative because the underlying use incompatibility remains.

- **Form-based mistaken for Performance.** A code emphasizes walkability and mixed use through form requirements (frontage, fenestration) but does not tie approval to measurable thresholds — a developer must still obtain discretionary approval based on design compatibility. The tell is that the form standard is aspirational or subject to variance. Consequence: slower approval, more subjective review, inconsistent outcomes.

- **Inclusionary zoning as a standalone zoning type.** Inclusionary zoning is almost always an **overlay** on top of Euclidean, form-based, or performance zoning, not a base regime. Misclassifying it as the primary zoning type obscures what development pattern is actually allowed. Consequence: developers and planners disagree on what the underlying zoning permits.

- **Performance zoning without capacity.** A jurisdiction adopts performance standards but lacks technical staff or consultant budget to review traffic models, stormwater calculations, or environmental impact assessments. The tell is that permits are issued slowly, consultants push back, or thresholds are routinely waived. Consequence: performance zoning devolves into slow Euclidean-style variance process.

- **Treating inclusionary zoning as affordability assurance.** A mandate for 15% of units at 80% AMI does not ensure long-term affordability if there is no restriction length (perpetual vs. 30 years) or enforcement mechanism (deed restriction, CLT, public purchase option). Misclassifying loose inclusionary requirements as equivalent to strong ones overestimates equity impact. Consequence: units become market-rate after restriction expires, creating false equity narrative.
