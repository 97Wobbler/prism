# Is-Ought Problem (Hume)

## The Rule
You cannot derive a normative claim (what ought to be) from a purely descriptive claim (what is) without smuggling in an unstated value premise.

## When It Applies
- A policy recommendation is presented as the inevitable conclusion of observed facts, without stating the values that make the recommendation follow.
- An empirical finding about human behavior (people *do* X) is cited as evidence that people *should* do X.
- A technical analysis concludes with a design decision framed as if the facts alone dictate it, when competing values are actually doing the work.
- A business case argues that market conditions *require* a particular strategy without acknowledging that other strategies serve different stakeholder interests equally well.

## When It Misleads
- As a blocker on all practical reasoning. Every useful decision smuggles in *some* value; the heuristic is a diagnostic tool, not a veto on decision-making.
- Applied as skepticism toward domain experts whose values are implicit because they are shared within a community. A structural engineer's recommendation contains hidden values (safety margins, cost tolerance, aesthetics) but is not thereby invalid.
- When conflating "value premise is unstated" with "value premise is unjustified." Some values are so widely held they need not be restated in every argument; the gap is real but not always a defect.
- In contexts where the audience shares a common goal, restating the value premise on every claim becomes noise rather than clarity.

## Common Misuse
Using the is-ought gap as an argument ender: "Well, you can't derive that from facts alone, so your recommendation is invalid." The real work is identifying what value premise is in play and whether it is one the audience endorses. Pointing out the gap is only the first step.

Also common: confusing "this value premise is implicit" with "this value premise is hidden or deceptive." Many good arguments work with shared assumptions; the heuristic is to make them explicit, not to assume they are being concealed.

## How Agents Use It
- **Embedded**: in policy analysis or recommendation lenses, as a mandatory step after deriving conclusions from data. For each recommendation, state the value premise (efficiency, equity, safety, consent, etc.) that bridges the facts to the ought.
- **Sanity-gate**: on each finding that concludes with a "should" or "must," ask: "What values does this conclusion assume, and are they ones the decision-maker actually holds?" If the gap is large, flag it for explicit negotiation rather than surprise.
