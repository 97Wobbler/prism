# Second Brain CODE

## Overview
The Second Brain CODE framework maps how information moves through four stages: Capture (collection), Organize (structure), Distill (refinement), and Express (output). Most knowledge systems fail not because they capture poorly, but because information decays between stages — organized but never distilled, distilled but never expressed. Practitioners use this lens to audit where their system leaks value and to design interventions that unblock the pipeline from input to decision or creation.

## Analytical Procedure

### Phase 1 — Map the Current Flow

1. **Choose an artifact to trace.** Pick a specific decision, project, or piece of writing you produced in the last 90 days. This becomes your test case.

2. **Identify the source material.** List every input that fed into the artifact — books, conversations, articles, notes, experiments, data, observations. Write down when each arrived and in what form (audio, text, video, verbal).

3. **For each source, trace its path through your system.**
   - **Capture stage:** Did you record it? How? (e.g., verbatim notes, recording, summary, highlight, or not at all?)
   - **Organize stage:** If recorded, where did it go? Did you tag it, link it, categorize it, or leave it loose?
   - **Distill stage:** Did you revisit it? Did you extract principles, synthesis, or implications? Or does it sit as raw capture?
   - **Express stage:** Did it appear in the artifact? If yes, how directly? If no, why not?

4. **Draw a flow diagram** (text or sketch). Use boxes for the four stages. Draw arrows showing how many sources made it to each stage. Sources that disappear at a stage are your leaks.

### Phase 2 — Quantify the Leakage

5. **Count the attrition.** For each source, mark it as:
   - **Captured:** Input was recorded in some form.
   - **Organized:** It was placed into a system (folder, database, collection) with retrieval structure (tags, links, categories).
   - **Distilled:** You actively refined it (wrote synthesis, extracted rules, challenged it, combined it with other sources).
   - **Expressed:** It appeared in the final artifact as evidence, reasoning, or content.

6. **Calculate retention at each stage.** 
   - Capture rate: (sources captured) / (total sources)
   - Organization rate: (sources organized) / (sources captured)
   - Distillation rate: (sources distilled) / (sources organized)
   - Expression rate: (sources expressed) / (sources distilled)
   - Overall: (sources expressed) / (total sources)

7. **Categorize each leak** by stage and reason:
   - **Capture leak:** Why wasn't it recorded? (Too fast, seemed unimportant, no tool at hand, distraction?)
   - **Organization leak:** Why wasn't it filed? (No clear category, unsure of relevance, procrastination?)
   - **Distillation leak:** Why wasn't it refined? (Too much raw capture, unclear how to condense, no review cycle?)
   - **Expression leak:** Why didn't it make it to the artifact? (Forgotten, not retrieved, seemed less relevant later?)

### Phase 3 — Identify Systemic Breaks

8. **Map tools and rituals to stages.** For each of the four stages, list:
   - The tool(s) you use (notes app, folder system, database, writing software, etc.)
   - The ritual or trigger that causes you to act (e.g., "review every Monday," "distill when a note hits 5 highlights," "pull references when writing").
   - The time cost of that ritual.

9. **Find disconnects.** Are tools designed for one stage but used for another? Example: Using a capture tool (raw notes) as your organization system (expecting to search it later) creates an invisible transition. Mark every tool mismatch.

10. **Identify missing rituals.** For each stage, ask: "How do I know when a piece of information is ready to move to the next stage?" If the answer is "I'm not sure" or "I just feel like it," that stage has no gate. Missing gates are where leaks hide.

### Phase 4 — Evaluate the Distillation Bottleneck

11. **Sample 10 organized sources.** Pick 10 pieces from your organized collection that are older than 30 days.

12. **For each, answer:**
   - Is it raw or distilled? (Raw = verbatim capture; Distilled = synthesis, reduction, reframing)
   - If raw: What would distillation look like? (1 sentence summary? comparison to other ideas? a principle? a caveat?)
   - Time estimate: How long would it take to distill this one item?

13. **Identify the distillation impediment.** The most common are:
   - **Surfacing:** Sources stay invisible. You forget they exist or can't find them.
   - **Clarity:** Sources are there but you're unsure what they mean or why they matter.
   - **Synthesis:** You have sources but don't know how to combine or compare them.
   - **Conviction:** You've extracted the idea but aren't confident enough to condense it.

## Evaluation Criteria

| Check | Pass |
|---|---|
| Source list is complete (all major inputs are named) | Y/N |
| Flow diagram shows all four stages with attrition counts | Y/N |
| Retention rates calculated for each stage and overall | Y/N |
| Each leak is categorized by stage and reason | Y/N |
| Tools mapped to stages with time cost noted | Y/N |
| At least one tool mismatch identified | Y/N |
| Missing rituals found (stages without advancement gates) | Y/N |
| 10-item distillation sample analyzed | Y/N |
| Distillation impediment is named and concrete | Y/N |

## Red Flags

- Capture rate is under 40%. You are losing information before it enters your system. No downstream intervention will help; the capture phase needs redesign.
- Organization rate is high but distillation rate is under 20%. You have a surfacing/clarity problem — sources exist but are not being refined. The issue is not tools but ritual.
- All tools are general-purpose (one note app for all four stages). Collapse across stages creates invisible transitions and is a common reason for distillation failure.
- No rituals exist. You rely on "when I remember" or "when it feels right." This is not a system; this is hope. Every stage needs a trigger.
- Distillation is described as "optional" or "nice to have." Distillation is where raw information becomes usable. Skip it and you are storing, not thinking.
- Expression rate is 100%. Either your curation is perfect (unlikely) or you are not being selective — everything you organize eventually gets used, which suggests you are not challenging sources enough during distillation.
