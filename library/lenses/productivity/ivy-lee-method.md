# Ivy Lee Method

## Overview

The Ivy Lee Method fixes a task list to exactly six items per day, ranked in priority order, and enforces serial execution: do nothing else until task 1 is complete, then move to task 2. The discipline is the constraint — most to-do systems collapse under ambition, allowing the practitioner to feel productive while completing nothing of value. Practitioners use this when their task list has become a mirror of anxiety rather than a plan, or when they suspect they are optimizing for activity count rather than outcome quality.

## Analytical Procedure

### Phase 1 — Audit Current Practice

1. **Retrieve your actual task list from the past week.** Not your ideal list — your real one. Copy it from your calendar, email, notes, or memory. Write down all open tasks you touched, attempted, or thought about.

2. **Count the tasks.** How many items are on the list? Flag if the number is more than 12 — this is the "false urgency" signal that the method is designed to catch.

3. **Identify which tasks you actually completed.** Mark each one. For tasks you did not complete, note why: Did you deprioritize it mid-week? Did a new task interrupt? Did you misjudge effort?

4. **Rank the incomplete tasks by stated importance.** Ask yourself: "If I could only finish three of these before Friday, which three matter most to my actual goals (not just urgency)?" Write the ranking. Do NOT rely on how the tasks feel; use outcome consequence.

### Phase 2 — Apply the Six-Task Constraint

5. **Select exactly six tasks for tomorrow.** Not "up to six." Exactly six. This forces a real choice. If you cannot identify six tasks of meaningful consequence, your goal clarity is poor — note this.

6. **Rank them 1–6 by outcome value.** Use this rubric:
   - **Rank 1:** If I complete nothing else this week, this task must be done. It is on the critical path to a real outcome.
   - **Rank 2–3:** These tasks unblock or enable rank 1. Or they deliver value on their own.
   - **Rank 4–6:** These are maintenance or nice-to-have. They are *not* excuses to abandon rank 1 if unexpected work arrives.
   If you cannot articulate why a task is ranked lower, it should not be on the list.

7. **Write the list in physical form or high-visibility digital place.** The visibility enforces the constraint. Low visibility (buried in an app) undermines the method — the list will expand invisibly.

8. **Execute serially.** Start task 1. Do not open email, Slack, or other task sources until task 1 is complete or the day ends. Move to task 2 only after task 1 is genuinely done. If an interruption arrives, evaluate it against rank 1: Does it displace rank 1? If no, defer it to tomorrow's list.

9. **At end of day, evaluate completion and reason.** Did you finish all six? If not, which ones remained and why? Was the ranking wrong, or was effort estimation wrong, or were interruptions legitimate?

10. **Carry incomplete tasks forward only if they are rank 1 or 2.** Anything ranked 4–6 that didn't finish is discarded. It was not actually essential. If it was essential, it was mis-ranked.

### Phase 3 — Measure Constraint Adherence

11. **Track for one week.** Record daily: task count, completion rate, interruptions that displaced rank-1, and tasks carried forward. This data reveals whether you are using the method or merely writing lists.

12. **Compare week-over-week completion on rank-1 and rank-2 tasks.** The method's value is not "I finished more tasks." It is "I finished the tasks that actually matter." A week where you completed 3 of 6 tasks but both of them were rank 1 is a success. A week where you completed 8 of 8 tasks but none were ranked above 3 is failure.

## Evaluation Criteria

| Check | Pass |
|---|---|
| Exactly six tasks selected (not "about six" or "up to six") | Y/N |
| Each task is ranked 1–6 with explicit consequence for ranking | Y/N |
| Rank-1 task has a specific outcome, not a vague goal | Y/N |
| No task below rank 4 is labeled "critical" or "urgent" | Y/N |
| Serial execution was attempted (one task at a time) | Y/N |
| Incomplete tasks were evaluated for rank before carry-forward | Y/N |
| Completion tracked for minimum 3 days | Y/N |

## Red Flags

- Task count creeps above six by mid-week. The practitioner is abandoning the constraint under pressure, which defeats the entire point.
- All six tasks are ranked "critical." This is not ranking; it is capitulation. The method requires hard choice.
- Rank-1 task is still incomplete at end of day but rank-4 or rank-5 task is marked done. Either the ranking was wrong or execution discipline failed. Either way, the method is not being applied.
- No tasks are carried forward incomplete. Either the original list was trivial (too short, too easy) or the practitioner is discarding high-value incomplete work to make the daily record look good.
- List contains both "finish report" and "email Sarah about report" as separate tasks. Granularity is too fine. The method requires tasks that take hours to days, not minutes. Sub-task bundling hides the real constraint.
- Task descriptions are vague ("work on project X," "respond to feedback"). Without specific outcomes, ranking is impossible and completion is unmeasurable.
