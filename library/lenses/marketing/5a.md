# 5A (Kotler)

## Overview

The 5A framework structures customer progression from initial awareness through active advocacy as five distinct stages: Aware, Appeal, Ask, Act, and Advocate. Each stage represents a measurable shift in customer state and requires targeted interventions. Practitioners use this lens to audit whether a marketing campaign or product launch addresses all conversion gates and to locate stage-specific bottlenecks where prospects stall. The discipline is mapping every intervention in the customer journey to one of these five buckets — vague activities ("build engagement") are red flags.

## Analytical Procedure

### Phase 1 — Map the Current Customer Journey

1. **List all marketing and product touchpoints for your target customer.** Include: advertising, content, email, website pages, sales conversations, onboarding, product features, support interactions, community spaces. Write them in the order a customer typically encounters them.

2. **For each touchpoint, write its stated goal.** Examples: "Blog post educates about the problem," "Landing page converts readers to leads," "Demo call addresses objections." Be literal — what does this touchpoint claim to accomplish?

3. **Classify each touchpoint to one 5A stage:**
   - **Aware:** Customer learns the problem or solution exists (ads, content, reviews, word-of-mouth)
   - **Appeal:** Customer finds the offer compelling vs. alternatives (comparison content, testimonials, pricing visibility)
   - **Ask:** Customer signals interest and seeks more information (email signup, form submission, sales inquiry, trial request)
   - **Act:** Customer makes a purchase or commits a resource (payment, account creation, contract signature)
   - **Advocate:** Customer recommends to others or publicly endorses (referral, review, testimonial, community participation)

   Write the stage next to each touchpoint. If a touchpoint serves multiple stages, list all.

### Phase 2 — Identify Stage Coverage Gaps

4. **Count touchpoints per stage.** Create a table:
   | Stage | # Touchpoints | Examples |
   |---|---|---|
   | Aware | | |
   | Appeal | | |
   | Ask | | |
   | Act | | |
   | Advocate | | |

5. **Flag underrepresented stages.** If any stage has fewer than 2 touchpoints, or if one stage has 3+ times more touchpoints than another, note it as a gap.

6. **For each gap, ask: "Why is this stage neglected?"** Answers might be:
   - Resource constraint (budget allocated elsewhere)
   - Assumption it's not needed ("people naturally advocate")
   - Measurement blindness (no tracking in place)
   - Structural absence (no product mechanism exists)
   
   Document the reason.

### Phase 3 — Audit Conversion Logic Between Stages

7. **Trace the path from one stage to the next.** For each pair (Aware→Appeal, Appeal→Ask, Ask→Act, Act→Advocate), write the explicit conversion trigger:
   - What causes a customer to move from Aware to Appeal? (Example: "Sees comparison table that ranks our solution first")
   - What causes a customer to move from Appeal to Ask? (Example: "Pricing is visible and not shocking; cares about feature X, which we have")
   - What causes a customer to move from Ask to Act? (Example: "Sales removes objections; trial works smoothly; payment is frictionless")
   - What causes a customer to move from Act to Advocate? (Example: "Product delivers promised outcome; we ask for a review; referral reward exists")

8. **For each trigger, verify that it's actually instrumented.** "We want people to see the comparison table" is not enough. Verify: Is the table present? Is it on the critical path? Is it measurable (impressions, time-on-page)?

### Phase 4 — Measure Stage Leakage

9. **For each stage transition, write the current drop-off rate or conversion rate if known.** Example: "80% of visitors reach Aware, 20% reach Appeal, 8% reach Ask, 3% reach Act, <1% reach Advocate."

10. **Identify the steepest cliff.** Which transition loses the most customers? This is the highest-leverage fix point. Document it.

11. **Hypothesize a root cause for the largest cliff.** Is it:
    - Messaging misalignment (Appeal doesn't match what Aware learned)?
    - Friction (Ask requires too much effort)?
    - Motivation gap (customer doesn't yet believe the solution works)?
    - Structural (no Ask mechanism exists)?

## Evaluation Criteria

| Check | Pass |
|---|---|
| All marketing and product touchpoints are inventoried (not just ads or email) | Y/N |
| Each touchpoint is assigned to at least one 5A stage | Y/N |
| Coverage table shows explicit counts per stage | Y/N |
| At least one underrepresented stage is identified | Y/N |
| Conversion trigger between each pair of stages is documented | Y/N |
| Each trigger is verified as present/measurable, not just aspired | Y/N |
| Stage transition data (conversion rates, drop-off) is cited | Y/N |
| Largest cliff is identified and a root cause is hypothesized | Y/N |

## Red Flags

- Touchpoints are listed only for Aware and Act, with Act unexpectedly high. Likely meaning: Ask/Appeal are not being tracked or they're conflated with Aware.
- Every touchpoint claims to address "all five stages." This means the framework was not applied — stages were not distinguished.
- Advocate stage has no touchpoints. If the product is working and customers are satisfied, Advocate must be explicitly designed; it does not happen by default.
- Conversion rates are unknown ("we don't track it"). The lens cannot work without data. Measurement is a prerequisite, not an afterthought.
- The "root cause" of a cliff is stated as "our messaging is bad" or "we need better engagement." Vague diagnoses. Go deeper: Which specific message? Which specific engagement mechanic?
- Only digital touchpoints are listed. Offline channels (calls, meetings, in-store, word-of-mouth networks) are omitted, which usually means they're either invisible or undervalued.
