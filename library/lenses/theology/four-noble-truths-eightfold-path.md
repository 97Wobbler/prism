# Four Noble Truths + Eightfold Path

## Overview

This framework pairs diagnostic clarity with a graduated prescription. The Four Noble Truths identify suffering, locate its cause, declare its cessation possible, and name the path to cessation. The Eightfold Path operationalizes that path through eight interdependent practices, grouped into three domains: wisdom, ethical conduct, and mental discipline. Practitioners use this when a problem feels systemic (not just a symptom) and they suspect the root cause is structural rather than accidental. The framework forces you to separate diagnosis from treatment and to design the treatment as a coherent sequence, not isolated fixes.

## Analytical Procedure

### Phase 1 — Diagnosis (Four Noble Truths)

1. **State the suffering clearly.** Describe the actual experienced pain, dysfunction, or failure mode in concrete terms. Not "the team is dysfunctional" but "sprint planning takes 8 hours, decisions are reversed mid-sprint, and 40% of committed work slips each cycle."

2. **Identify the origin (First Truth: Suffering exists).** Document the suffering across at least three dimensions:
   - What breaks or fails? (outcome)
   - How often and at what cost? (frequency and impact)
   - Who experiences it most acutely? (stakeholder specificity)

3. **Locate the root cause (Second Truth: Suffering has a cause).** The cause is usually NOT the symptom. Ask: "If I had to explain this to someone who knew nothing about our context, what precondition, belief, incentive, or structure would I have to describe for them to understand why this breaks?"
   - Look for misaligned incentives (people rewarded for conflicting goals)
   - Look for information asymmetries (some parties lack data others have)
   - Look for structural constraints (physical limits, dependencies, bottlenecks)
   - Look for unexamined assumptions (we've always done X, so we assume X is necessary)
   Do not confuse root cause with proximate cause. "We didn't plan well" is proximate. "Planning requires consensus from five teams with no shared metrics" is closer to root.

4. **Declare the possibility of cessation (Third Truth: Suffering can cease).** Before designing a remedy, establish that the problem is not metaphysically permanent. State what the system would look like if the root cause were removed:
   - Specify the end state in observable terms (not "better" but "sprint planning completes in 2 hours, 90% of committed work ships, zero mid-sprint reversals")
   - Identify at least one real-world system where this end state exists (do not invent; find a comparable domain or organization)
   - Estimate the cost of reaching that state (rough order of magnitude)

### Phase 2 — Prescription (Eightfold Path)

The path has eight practices organized into three domains. Each practice is a simultaneous action, not a sequence — they reinforce each other. However, the first practice in each domain is prerequisite to the others in that domain.

**Domain 1: Wisdom (Right View, Right Intention)**

5. **Right View — Diagnose the root cause frame.** Develop a shared, evidence-based understanding of why the problem exists. This is not blame; it's seeing the system as it actually is. Create an artifact:
   - One-page memo explaining the root cause to a newcomer
   - Include the structural or incentive trap that generates the suffering
   - Include evidence (data, quotes, observed patterns) that supports the diagnosis
   - Seek explicit agreement from stakeholders that this frame is accurate before proceeding

6. **Right Intention — Align intention with outcome.** State the intention behind the remediation:
   - What does success look like for each stakeholder?
   - Are these intentions compatible, or do they conflict?
   - If they conflict, which takes priority and why?
   - State the intention in terms of what you are moving *toward*, not what you are escaping *from*

**Domain 2: Ethical Conduct (Right Speech, Right Action, Right Livelihood)**

7. **Right Speech — Establish norms for how change will be discussed.** During remediation, conflict will surface. Pre-agree on how disagreement will be handled:
   - Will dissent be welcomed or suppressed?
   - How will trade-offs be communicated?
   - Who has authority to make decisions, and by what process?
   - What is explicitly off-limits to say (disrespect, contempt)?
   Document this as a brief covenant. Revisit it if violations occur.

8. **Right Action — Design concrete, reversible interventions.** Do not attempt to fix the entire root cause at once. Break the path into 3–5 discrete interventions:
   - Each intervention should be small enough to implement and measure in 2–4 weeks
   - Each should be reversible (if it fails, you can undo it without cascading damage)
   - Each should remove or reduce one aspect of the root cause
   - Order them so earlier wins build credibility and give psychological momentum for later, harder changes
   Example: If the root cause is "five teams with no shared metrics," interventions might be: (1) agree on a shared KPI definition, (2) create a shared dashboard, (3) align incentive metrics to shared KPI, (4) restructure standup to use shared KPI as agenda, (5) review and iterate the KPI quarterly.

9. **Right Livelihood — Ensure the remediation is sustainable.** For each intervention:
   - Who maintains it after launch?
   - What resources (time, tools, headcount) does maintenance require?
   - Is that sustainable given current staffing and priorities?
   - If not, redesign the intervention to be less burdensome or allocate new resources before launch

**Domain 3: Mental Discipline (Right Effort, Right Mindfulness, Right Concentration)**

10. **Right Effort — Commit to iteration and non-negotiable discipline.** Declare:
    - How often will you measure progress against your end-state vision?
    - What will you do if progress stalls?
    - What will trigger a pivot or escalation?
    - Who has authority to call for a pause, pivot, or restart?

11. **Right Mindfulness — Maintain awareness of unintended effects.** As you execute interventions:
    - What side effects are you observing?
    - Are any interventions harming stakeholders you did not anticipate?
    - Is the root cause shifting or metastasizing into new forms?
    - Maintain a visible log of these observations (not hidden in Slack; visible to the team)

12. **Right Concentration — Protect focus.** Prevent the remediation from being derailed by:
    - Scope creep (new problems surfacing that are not part of the root cause)
    - Parallel initiatives that undermine the path
    - Distraction by quick fixes that feel urgent but don't address the root cause
    Make a written list of things you will *not* attempt during this remediation. Revisit it monthly.

## Evaluation Criteria

| Check | Pass |
|---|---|
| Suffering is stated in concrete, measurable terms | Y/N |
| Root cause is structural/systemic, not proximate | Y/N |
| End state is specified in observable terms | Y/N |
| At least one real-world precedent exists for the end state | Y/N |
| Right View memo is clear enough for a newcomer to understand | Y/N |
| Intentions of all major stakeholders are listed and conflicts surfaced | Y/N |
| Speech norms are pre-agreed and documented | Y/N |
| Interventions are discrete, reversible, and sequenced | Y/N |
| Each intervention has a clear owner and maintenance plan | Y/N |
| Iteration cadence and decision authority are defined | Y/N |
| Side-effect log is visible and reviewed regularly | Y/N |
| "Do not attempt" list is maintained and protected | Y/N |

## Red Flags

- Root cause is stated as a behavior or symptom ("the team doesn't collaborate") rather than a structure ("no shared metric means each team optimizes locally, creating misalignment").
- End state is aspirational ("we'll be agile and happy") rather than measurable ("sprint planning 2 hours, 90% commitment delivery").
- The Eightfold Path is treated as a checklist instead of a reinforcing system. If interventions are not interdependent and mutually supporting, the prescription is likely incomplete.
- Stakeholder intentions are not explicitly surfaced or conflicts are papered over ("everyone wants what's best for the team"). This always surfaces later as sabotage.
- Interventions are attempted all at once. Parallel changes destroy the ability to measure what actually worked.
- No reversibility clauses. If you cannot undo an intervention without organizational damage, it is too radical for this framework; it needs more design.
- Side-effect log exists but is not reviewed. Unintended harms accumulate invisibly.
- "Do not attempt" list is absent or ignored. Scope creep is the #1 reason remediation paths fail.
