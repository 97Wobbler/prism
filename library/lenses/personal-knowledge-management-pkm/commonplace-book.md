# Commonplace Book (Locke)

## Overview

A commonplace book is a system for capturing, organizing, and retrieving excerpts, observations, and ideas by building a searchable personal index rather than storing them by source or chronology. Instead of collecting quotes and insights in a notebook in order of discovery (where retrieval depends on remembering when you read it), you capture each item with minimal metadata and create a thematic index that allows you to find related ideas across sources. Practitioners use this when they want to prevent insights from evaporating, build a personal reference system, or discover connections between ideas read months or years apart.

## Analytical Procedure

### Phase 1 — Establish Capture Discipline

1. **Define what counts as a capture.** Decide your unit of entry: a full quote, a paraphrase, a conceptual insight, or a reaction. Write down your rule explicitly. Example: "I capture anything that makes me pause or disagree." Do not change this rule mid-project — consistency across time is what makes the index useful.

2. **Select or create a container.** This can be a physical notebook, digital document, spreadsheet, wiki, or note app. The container must allow you to:
   - Add dated entries with minimal friction
   - Tag or label each entry with concept names
   - Search entries by both tag and free text
   - Export or print the full collection without reformatting

3. **Establish a metadata template.** For each capture, record:
   - **Source**: Author, book title, publication year (or URL + date accessed)
   - **Location**: Page number, chapter, URL section, or timestamp (enables returning to context)
   - **Excerpt or observation**: The exact words or your paraphrase, clearly marked which
   - **Concept tags**: 2–5 thematic labels (e.g., "decision-making," "risk," "bureaucracy")
   - **Personal reaction** (optional): Why you captured it; your disagreement or follow-up question

   Example:
   ```
   Source: Simon, Herbert (1956). "The Machinery of the Mind." Scientific American.
   Location: p. 71
   Excerpt: "The capacity of the human mind for formulating and solving complex problems is very small compared with the objective complexity of the problems that scientists face."
   Tags: cognitive-limits, bounded-rationality, decision-making
   Reaction: Explains why we simplify problems before solving them — a feature, not a bug.
   ```

### Phase 2 — Build the Index

4. **As entries accumulate (minimum 20–30), audit your tags.** Review all concept labels you've used. Identify:
   - **Orphans**: Tags used only once (likely too specific; merge into a broader concept)
   - **Clusters**: Three or more entries sharing the same tag (strong concept, worth centralizing)
   - **Synonyms**: Different names for the same idea (e.g., "risk," "uncertainty," "downside")

5. **Consolidate and standardize tags.** Merge synonyms, eliminate orphans, rename for consistency. Example: if you have "decision-making," "choosing," and "choice," pick one and retag all entries.

6. **Create a two-level index structure.** At minimum, produce:
   - **Primary index**: A list of your final concept tags in alphabetical order
   - **Concept pages**: For each tag, list every entry tagged with it, sorted by date or by which you want to encounter them (e.g., foundational sources first)

   Example structure:
   ```
   ## Concept: Bounded Rationality
   
   1. Simon (1956) — The capacity of the human mind for formulating...
      [Link or page number to full entry]
   2. Kahneman (2011) — System 1 thinking is fast but error-prone...
   3. Thaler (2015) — People are predictably irrational in ways that...
   ```

### Phase 3 — Use the Index

7. **When you want to explore a topic, query the index by concept, not by source.** Rather than "What did I read about economics?", ask "All my entries on decision-making" and follow the index. This surfaces connections across sources you didn't consciously make.

8. **Regularly review entries under a single concept tag.** Reading your collected excerpts on (say) "communication" or "organizational design" in sequence reveals patterns you would miss reading them scattered across books. Record any synthesis or new insight as a new entry tagged with the concept plus a note like "synthesis from 3+ sources."

9. **Periodically audit for decay.** Every 3–6 months, check:
   - Are there tags with 0 entries? Consider deleting them.
   - Are there entry clusters with 10+ items? Consider splitting into subtags (e.g., "decision-making" → "decision-making: individual" + "decision-making: organizational").
   - Are there entries you've never revisited? Either they belong in a different concept or they aren't useful to you; be honest and prune.

## Evaluation Criteria

| Check | Pass |
|---|---|
| Capture rule is explicit and documented | Y/N |
| Every entry has source, location, excerpt, and ≥2 concept tags | Y/N |
| Tags are standardized (no synonyms; orphans merged) | Y/N |
| Index is organized by concept, not chronology or source | Y/N |
| Index is searchable (by tag and by text, if digital) | Y/N |
| At least one synthesis across 2+ sources exists | Y/N |
| The system was actively used for ≥3 months | Y/N |

## Red Flags

- Entries accumulate faster than the index grows. You're capturing but not indexing — the entries are invisible to you and act as dead weight.
- Most entries are single-tagged or have 8+ tags. Single-tag suggests your concepts are too broad; 8+ suggests you're uncertain about what each entry is actually about.
- Searches in the index reliably return 0 or 1 matches. Either your tags don't align with how you think, or you're not using it.
- Tags like "miscellaneous," "quotes," "important," or "to-sort" appear. These are procedural labels, not concepts — they indicate the index wasn't built.
- The notebook is full but you've never re-read a single concept page. A commonplace book whose entries are never reviewed is just a graveyard.
