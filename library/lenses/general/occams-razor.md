# Occam's Razor

## Overview
When multiple explanations fit the same evidence, prefer the one that requires the fewest assumptions. The point is not "the simplest wins" — that's a caricature. The point is to make the assumption count *explicit and comparable*, because people routinely defend elaborate explanations whose hidden assumptions they've never enumerated. Practitioners use it to break tie-votes between hypotheses and to catch over-engineering.

## Analytical Procedure

1. **List the competing explanations or solutions (min 2, max 5).** Name each one. Vague alternatives ("or maybe something else") don't count.

2. **For each explanation, enumerate its assumptions.** An assumption is anything the explanation requires to be true but isn't directly proven by the evidence. Write them as atomic, verifiable claims:
   - Bad: "The network is weird."
   - Good: "Assumption A1: DNS resolution for internal hosts takes longer than 50ms under load."
   Aim for exhaustive — missing assumptions is how Occam's Razor is misapplied.

3. **Tag each assumption by type:**
   - **Empirical** — can be tested with data we already have or can cheaply collect
   - **Structural** — requires the world/system to be configured in a specific way we haven't verified
   - **Causal** — asserts that X causes Y, not just that they co-occur
   - **Motivational** — asserts that a person/group acted with a specific intent
   Motivational and causal assumptions are the most expensive — they're hardest to verify and easiest to smuggle in.

4. **Count and weight.** Raw count is the starting signal, but a single causal assumption is "heavier" than three empirical ones. Produce two numbers per explanation: **raw count** and **weighted count** (empirical=1, structural=2, causal=3, motivational=3).

5. **Check that each explanation actually fits the evidence equally well.** Occam's Razor only applies between explanations that are empirically equivalent. If Explanation A covers 95% of the evidence and Explanation B covers 60%, A wins regardless of assumption count — note this and stop applying the razor.

6. **Select the explanation with the lowest weighted count among empirically equivalent candidates.** State explicitly which assumptions of the losing explanations you are now committed to disbelieving.

7. **Pre-register the falsifier.** For the winning explanation, name the single observation that would most cheaply disprove it. If no such observation exists, the explanation is not actually scientific — flag it.

## Evaluation Criteria

| Check | Pass |
|---|---|
| At least 2 explicit competing explanations | Y/N |
| Each explanation has ≥3 enumerated assumptions | Y/N |
| Each assumption is typed (empirical/structural/causal/motivational) | Y/N |
| Empirical equivalence of explanations is checked before applying the razor | Y/N |
| Weighted count is produced, not just raw count | Y/N |
| A cheap falsifier is stated for the winning explanation | Y/N |

## Red Flags

- One explanation has "0 assumptions." No explanation is assumption-free — this means assumptions are hidden, not absent.
- The "simpler" explanation fits the evidence noticeably worse and is being chosen anyway. That's parsimony abuse.
- Motivational assumptions dominate ("they did it because they wanted to..."). These are nearly impossible to verify and should rarely be load-bearing.
- The list of explanations was generated by one person without adversarial input. High risk of a missing alternative the razor never got to evaluate.
- The "winning" explanation has no falsifier. It's unfalsifiable, so the razor didn't actually do work.

```
## Occam's Razor Assessment

**Evidence being explained:**
<bullet list of observations all explanations must account for>

### Explanation A: <name>
- Fits evidence: <% or qualitative>
- Assumptions:
  - A1 [empirical]: <claim>
  - A2 [causal]: <claim>
  - ...
- Raw count: _  |  Weighted count: _

### Explanation B: <name>
- Fits evidence: <% or qualitative>
- Assumptions:
  - B1 [structural]: <claim>
  - ...
- Raw count: _  |  Weighted count: _

### Empirical Equivalence Check
<explanations being compared are empirically equivalent | one dominates — razor not applied>

### Selection
- Winner: <A | B | ...>
- Reason: <lowest weighted count among equivalents>
- Commitments implied: <which losing assumptions we now disbelieve>

### Falsifier
- Cheapest observation that would disprove the winner: <specific test>

### Confidence
<high/medium/low> — <justification>
```
