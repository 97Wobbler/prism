# Disney Method

## Overview
The Disney Method structures creative ideation across three distinct roles — Dreamer (unconstrained imagination), Realist (practical implementation), and Critic (rigorous interrogation) — applied sequentially to a single problem or proposal. Practitioners rotate through these perspectives deliberately, resisting the urge to collapse them into a single conflicted viewpoint. Use this when a team is stuck between "that will never work" and "anything is possible," or when a single voice dominates and others cannot articulate their concerns clearly.

## Analytical Procedure

### Setup

1. **Define the creative challenge in one sentence.** No jargon. Examples: "How do we reduce customer onboarding time?" or "What should our new product feature do?"

2. **Assemble the group physically or virtually.** Minimum: one person per role. Larger groups divide into three sub-teams. Maximum effectiveness with 3–12 people total.

3. **Designate a space (physical or virtual) for each role.** If physical, use three tables or corners. If virtual, use three breakout rooms or clearly labeled sections. This spatial separation is intentional — it makes role-switching feel distinct and serious.

4. **Set a time box for each role phase.** Typically 15–30 minutes per role, depending on complexity. Announce it upfront.

### Phase 1 — Dreamer

5. **Move to the Dreamer space.** Dreamer rules: there are no constraints. Ignore budget, technology, regulation, and feasibility. The goal is quantity and novelty, not quality.

6. **Generate ideas without judgment.** Use prompts like:
   - "What if money were unlimited?"
   - "What if we had a magical tool that could do anything?"
   - "What would the ideal future state look like?"
   - "What would delight a user beyond what they expect?"

7. **Write down every idea, no matter how wild.** Do not evaluate. Do not edit. Do not say "that's not realistic." Assign someone to scribe. Aim for 20+ ideas, not 5 polished ones.

8. **At the end of Phase 1, read the list aloud.** Dreamer says: "These are our possibilities." Do not defend or explain yet.

### Phase 2 — Realist

9. **Move to the Realist space.** Realist rules: assume all Dreamer ideas are worth implementing. The job is to build a concrete path, not kill ideas.

10. **For each Dreamer idea (or the top 3–5 if the list is long), answer:**
    - How would we actually do this? (Steps, tools, people required)
    - What is the timeline?
    - What are the dependencies?
    - What is the rough cost in time/money/effort?
    - Who owns each step?

11. **Build execution plans, not critiques.** Example: instead of "that's too expensive," say "To do X, we need 2 engineers for 6 weeks. That costs $Y. We can stage it: Phase 1 (2 weeks), Phase 2 (4 weeks)."

12. **Make assumptions explicit but don't derail on them.** Write: "Assumes we can hire 2 engineers by Month 2" or "Assumes API stability."

13. **At the end of Phase 2, Realist says: "Here's how we'd make it real."** Present the plans. Do not say "this won't work" — instead describe what *would* need to be true for it to work.

### Phase 3 — Critic

14. **Move to the Critic space.** Critic rules: the job is to find weaknesses in the Realist plans, not to veto Dreamer ideas. Critic interrogates *how*, not *whether*.

15. **For each Realist plan, ask:**
    - What could go wrong? (Technical, organizational, market, user adoption)
    - What assumptions are weakest?
    - What is the highest-leverage risk? (i.e., what breaks everything if it fails?)
    - What are we not seeing? (blind spots, adjacent problems)
    - Does this solve the original challenge or just feel good?
    - Are there internal contradictions? (Realist plan requires X, but also assumes Y, which conflicts.)

16. **Write findings as structured risks, not as dismissals.** Example: instead of "users won't adopt it," write "Risk: User adoption if training is not available. Mitigation: plan user education in Phase 2."

17. **Score each plan on:**
    - Feasibility (can it actually be built given stated constraints?)
    - Relevance (does it address the original challenge?)
    - Robustness (will it survive contact with reality?)

    Use a simple scale: High / Medium / Low.

18. **At the end of Phase 3, Critic says: "Here's what we need to watch."** Present risks and the most robust plans. Do not conclude "don't do this."

### Phase 4 — Synthesis

19. **Return to shared space.** Moderator recaps:
    - Dreamer's best ideas
    - Realist's most feasible plans
    - Critic's highest-leverage risks

20. **Decide whether to:**
    - **Proceed** with the highest-scoring plan and assign owners (Realist step 13).
    - **Iterate:** Return to Dreamer or Realist to refine weak points the Critic identified.
    - **Reject:** If Critic finds a fatal flaw (e.g., violates a non-negotiable constraint), name it and start over with a new Dreamer prompt.

21. **Document:** Record the three phases, the final decision, and the owners. This becomes the shared brief for execution.

## Evaluation Criteria

| Check | Pass |
|---|---|
| Challenge defined in one sentence | Y/N |
| All three roles were enacted (not skipped) | Y/N |
| Dreamer phase produced 15+ ideas with no evaluation | Y/N |
| Realist phase produced step-by-step plans with costs/timelines | Y/N |
| Critic phase identified specific risks with severity scores | Y/N |
| Assumptions were made explicit in Realist phase | Y/N |
| Critic did not veto; instead scored feasibility/relevance/robustness | Y/N |
| Synthesis step included explicit proceed/iterate/reject decision | Y/N |

## Red Flags

- All ideas in Dreamer phase are modest variants on the status quo. Either the Dreamer was not truly unconstrained or the facilitator interrupted.
- Realist plans are presented as critiques ("this is too expensive") instead of construction ("here's the path if we allocate X resources").
- Critic phase is dominated by one person's objections instead of systematic risk interrogation. Spread the analysis across the group.
- Roles collapsed: someone says "as a Dreamer I think we should be realistic" or "the real problem is..." Physically moving and time-boxing prevents this.
- No decision made at synthesis. The exercise ends in discussion instead of commitment. Assign an owner immediately or set a re-convene date.
- Critic phase is empty or produces only vague concerns ("it feels risky"). Push for specificity: which assumption? what breaks? what is the trigger?
